# Техническое задание (ТЗ) на игру **«Мышата»**  
**Команда:** Весёлые котики  

---

## 1. Общая характеристика приложения

Игра «Мышата» — это аркада, в которой игрок управляет мышонком, собирая сыр (монеты) и избегая опасностей: кошек, мышеловок и непроходимых стен. Игра содержит несколько окон:

- Начальное окно  
- Окно меню  
- Игровая часть  
- Окно окончания уровня  

---

## 2. Описание интерфейса

### 2.1. Начальное окно  
Отображается при запуске приложения. Содержит:

- Приветственное сообщение  
- Три кнопки:  
  - **«К уровням»** — переход к списку уровней  
  - **«Правила»** — открытие окна с правилами игры  
  - **«Управление»** — открытие окна с описанием управления  

#### Окно «Правила»  
Подробное описание:
- Цель игры — собрать как можно больше сыра за отведённое время  
- Опасности: кошки, мышеловки, стены  
- Последствия столкновений:  
  - С кошкой — +15 секунд к времени  
  - С мышеловкой — +10 секунд к времени  
- Прохождение уровня — сбор всех монет  

#### Окно «Управление»  
Описание управления:
- Перемещение мышонка — стрелки или WASD  
- Открытие меню во время игры — клавиша **Esc**  

#### Окно «К уровням»  
- Отображение всех уровней в виде сетки или списка  
- Пройденные уровни — выделены (например, зелёным цветом)  
- Непройденные уровни — заблокированы (серые, недоступны для выбора)  

---

### 2.2. Окно меню (внутриигровое)  

Доступно в любой момент игры по нажатию **Esc**. Содержит кнопки:

- **«Продолжить»** — скрыть меню, возобновить игру  
- **«Начать заново»** — перезапуск текущего уровня  
- **«Правила»** — открытие правил  
- **«Управление»** — открытие управления  
- **«Выход»** — возврат к списку уровней  

---

### 2.3. Игровая часть  

- Игрок управляет мышонком  
- На уровне разбросаны монеты (сыр) — необходимо собрать все  
- Препятствия:  
  - **Статичные мышеловки** — нельзя касаться  
  - **Двигающиеся кошки** — нельзя касаться  
  - **Непроходимые стены** — блокируют движение  
- При столкновении:  
  - С кошкой → +15 секунд к текущему времени  
  - С мышеловкой → +10 секунд к текущему времени  
- Цель — пройти уровень за минимальное время  

---

### 2.4. Окно окончания уровня  

Отображается после сбора всех монет. Содержит:

- Текущее время прохождения уровня  
- Лучшее (рекордное) время по этому уровню  
- Кнопки:  
  - **«Начать заново»** — перезапуск текущего уровня  
  - **«Выйти»** — возврат к списку уровней  
  - **«Дальше»** — переход к следующему уровню (если доступен)  

---

## 3. Основные функциональные требования

- Поддержка нескольких уровней с возрастающей сложностью  
- Сохранение лучших результатов по каждому уровню (локально)  
- Реализация коллизий между игроком и объектами (кошки, мышеловки, стены)  
- Таймер, отсчитывающий время прохождения уровня  
- Блокировка недоступных уровней до прохождения предыдущих  
- Полноценная навигация между экранами: начальное окно → уровни → игра → окончание → уровни  

---

## 4. Архитектура и дизайн

Приложение должно:
- Содержать спокойные тона
- Быть оформленно в Pixel Art стиле
- Иметь птнятную и логичную навигаю

## 5. Описание базы данных

База данных будет содержать таблицы с каждым уровнем (Level_1, Livel_2 ...) с переменными:
- Id (номер попытки)
- Time (Время прохождения с штрафными балами)

Каждый раз по окнончанию уровня должен записываться результтат.
